Points de design clés

Header sticky

  • Adresse cliquable · texte souligné discret + icône pin · tap ouvre Google Maps universel (Plans iOS / Maps Android)
  • Badge phase info bleu à côté de l'adresse · couleur sémantique (phase Esquisse = info, Chantier = info, Réception = success, etc.)
  • Menu kebab à droite · accès actions secondaires (édition fiche, archivage, partage projet)

CTA contextuel sticky

  • Bouton plein largeur terracotta · 1 seule action prioritaire par phase
  • Phase chantier → "Démarrer la visite" · permis → "Suivre le permis" · OPR → "Voir les réserves" · réception → "Préparer réception définitive"
  • Si rien de pertinent (esquisse, archivé) → le CTA disparaît, on gagne 60px d'écran utile

Tabs horizontaux scrollables

  • 9 onglets (Résumé/PV/Réserves/Permis/Journal/Photos/Documents/Tâches/Fiche) · scroll horizontal avec fade-out à droite pour indiquer la suite
  • Count badges par onglet · brand.100 si actif, neutral.100 sinon · l'archi voit d'un coup d'œil l'activité de chaque module
  • Dot d'alerte · point ambre 6px à côté du count si urgence (ex : Réserves a 2 en retard)
  • Underline 2px brand.500 sur l'actif · pas de fond plein

Tab content patterns

  • Résumé = TodoCard priority + grille 2×3 Outils du projet (réplique mobile de la vue ProjectDetail desktop wired commit ddf77b5)
  • Réserves / Photos / PV / Journal = filter chips + bouton "+" dashed in-context (pas un FAB qui chevaucherait le FAB Visite)
  • Bordure latérale gauche sur les réserves : 3px danger si en retard, 3px warning si majeure, rien sinon
